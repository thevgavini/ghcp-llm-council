About the Council

Five voices. Three rounds. One answer.

What this is

The Council is a GitHub Copilot CLI skill that runs every question past a panel of independent LLMs and watches them reason about each other's answers. Instead of trusting a single model, you get a deliberation — first opinions, peer review, and a final synthesis — rendered live in this browser pane while your CLI does the orchestration.

The mark above tells the story: 5 first opinions → 3 rounds of peer review → 1 chairman's synthesis.

How a question unfolds

  1. First opinions. Each councillor answers your question in parallel, without seeing the others.
  2. Peer review. The answers are anonymised and shown back to every councillor; each one ranks them and explains why.
  3. Chairman's synthesis. A single model reads the answers and the rankings, then writes the final response — informed by the council, accountable to one voice.

Backends

Two model backends are supported per councillor:

  • Copilot CLI — Anthropic and OpenAI models routed through the same task tool your agent already uses. No extra API keys.
  • GitHub Models — Meta Llama, DeepSeek, Mistral, Microsoft Phi, Cohere and others, called over HTTPS using your existing gh auth token.

Modes

Every question runs in one of five modes. The mode picks the prompt set the councillors and chairman work from, and may swap in a different lineup of models tuned for that question shape.

  • general Open-ended Q&A. The default. Five-vendor council, free-form synthesis. Use when no other mode fits.
  • review Code or change review. Critic-heavy four-model lineup. Councillors produce severity-tagged findings; chairman opens with a verdict (APPROVE / REQUEST CHANGES / BLOCK) and merges duplicate flags. Pair with --files to feed source directly.
  • design Architecture or technology choice. Each councillor enumerates 2–3 credible options, names trade-offs, and recommends one. Chairman writes a decision document with explicit "what would change my mind".
  • plan Implementation roadmap. Councillors produce ordered steps with rationale, risks, and a definition of done. Chairman picks the strongest ordering and merges the risk list.
  • research Learn about a topic. Councillors lead with a one-paragraph summary, then unpack mechanisms and numbers. Chairman synthesises an explainer with explicit confidence levels and caveats — no recommendation.

A pill appears next to non-general conversations in the sidebar and at the top of the canvas. The chairman and council composition can be customised per mode via mode_packs in defaults/council.json.
